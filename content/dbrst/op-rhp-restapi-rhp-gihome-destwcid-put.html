升级非托管Grid Infrastructure软件主页

放

/ RHP-RESTAPI / RHP / gihome / {destWcId}

将Oracle Grid Infrastructure从源主目录路径升级到目标工作副本。

请求

支持的媒体类型
  • 应用程序/ JSON
路径参数
  • destWcId：string
    需要移动Oracle软件的目标工作副本的名称
标题参数
  • 授权：字符串
    Base64编码用户名和发出请求的用户的密码。有关更多信息，请参阅Fleet Patching和Provisioning的身份验证 。
身体 （ ）
Root Schema：UpgradeGihomeBody
输入： object
显示来源
  • auth（可选）：字符串
    使用身份验证插件访问远程节点。
  • authArgs（可选）：对象 authArgs
    允许的其他属性： additionalProperties
    所选身份验证插件的其他属性。
  • destwc（可选）：字符串
    需要升级Oracle Grid Infrastructure的目标工作副本的名称。
  • eval（可选）：布尔值
    默认值： false
    无需执行命令即可进行评估。
  • ignoreprereq（可选）：布尔值
    默认值： false
    忽略CVU先决条件检查。
  • image（可选）：字符串
    配置图像中的图像名称。
  • path（可选）：string
    目标工作副本的绝对路径。
  • root（可选）：布尔值
    默认值： false
    使用root凭据访问远程节点。
  • schedule（可选）：string
    执行操作的首选时间，采用ISO-8601格式。例如：2016-12-21T19：13：17 + 05。
  • sourcehome（可选）：字符串
    源Oracle主路径。
  • sourcewc（可选）：字符串
    需要升级Oracle Grid Infrastructure的源工作副本的名称。
  • sudoPath（可选）：字符串
    sudo二进制文件的位置。
  • sudoUser（可选）：字符串
    以sudo用户名执行超级用户操作。
  • superUserPasswd（可选）：字符串
    超级用户帐户的密码。
  • targetnode（可选）：string
    没有Rapid Home Provisioning Client的远程群集中节点的名称。
{
    "type":"object",
    "properties":{
        "sourcewc":{
            "type":"string",
            "description":"Name of the source working copy from which the Oracle Grid Infrastructure needs to be upgraded."
        },
        "destwc":{
            "type":"string",
            "description":"Name of the destination working copy to which the Oracle Grid Infrastructure needs to be upgraded."
        },
        "image":{
            "type":"string",
            "description":"Image name from the configured images."
        },
        "targetnode":{
            "type":"string",
            "description":"Name of a node in a remote cluster with no Rapid Home Provisioning Client."
        },
        "sourcehome":{
            "type":"string",
            "description":"Source Oracle home path."
        },
        "ignoreprereq":{
            "type":"boolean",
            "default":false,
            "description":"To ignore the CVU pre-requisite checks."
        },
        "path":{
            "type":"string",
            "description":"The absolute path for the destination working copy."
        },
        "root":{
            "type":"boolean",
            "default":false,
            "description":"Use root credentials to access the remote node."
        },
        "sudoUser":{
            "type":"string",
            "description":"perform super user operations as sudo user name."
        },
        "sudoPath":{
            "type":"string",
            "description":"location of sudo binary."
        },
        "superUserPasswd":{
            "type":"string",
            "description":"The password for the super user account."
        },
        "authArgs":{
            "type":"object",
            "description":"Additional properties for the selected authentication plugin.",
            "additionalProperties":{
                "type":"string"
            }
        },
        "auth":{
            "type":"string",
            "description":"Use an authentication plugin to access the remote node."
        },
        "eval":{
            "type":"boolean",
            "default":false,
            "description":"Evaluate without executing the command."
        },
        "schedule":{
            "type":"string",
            "description":"Preferred time to execute the operation, in ISO-8601 format. For example: 2016-12-21T19:13:17+05."
        }
    }
}
嵌套架构：authArgs
输入： object
允许其他属性
显示来源

    "type":"object",
    "description":"Additional properties for the selected authentication plugin.",
    "additionalProperties":{
        "type":"string"
    }
}

安全

    类型： basic

支持的媒体类型
  • 应用程序/ JSON

202回应

公认。一个JSON对象，包含接受操作的作业资源详细信息，以获取进度的状态和输出。
头
  • 位置：字符串
    将执行操作的作业的URL
身体 （ ）
Root Schema：jobResource
输入： object
显示来源
  • jobId（可选）：字符串
  • links（可选）：数组 链接
  • operation（可选）：string
  • 输出（可选）：对象 jobOutputResource
  • status（可选）：string 
{
    "type":"object",
    "properties":{
        "jobId":{
            "type":"string"
        },
        "status":{
            "type":"string"
        },
        "operation":{
            "type":"string"
        },
        "output":{
            "$ref":"#/definitions/jobOutputResource"
        },
        "links":{
            "type":"array",
            "items":{
                "$ref":"#/definitions/singleResourceLinks"
            }
        }
    }
}

400回应

错误的请求。一个JSON对象，包含解析参数值或名称时发现的问题的错误详细信息。
身体 （ ）
Root Schema：errorResource
输入： object
显示来源
  • errorCode（可选）：字符串
    特定于应用程序的错误代码，与tittle中显示的错误消息有关。
  • errorDetails（可选）：array errorDetails
    如果报告了多个错误，则可以在此阵列中以分层结构组织它们。
  • errorPath（可选）：string
    用于指示错误发生位置的XPath或JSON路径。
  • instance（可选）：string
    链接的URI，提供有关错误的更多详细信息。
  • status（可选）：整数
    HTTP状态代码。
  • title（可选）：string
    摘要错误消息。
  • type（可选）：string
    链接到HTTP错误代码页。
{
    "type":"object",
    "properties":{
        "status":{
            "type":"integer",
            "description":"HTTP status code."
        },
        "type":{
            "type":"string",
            "description":"Link to HTTP error code page."
        },
        "title":{
            "type":"string",
            "description":"Summary error message."
        },
        "instance":{
            "type":"string",
            "description":"URI to the link that provides more detail about the error."
        },
        "errorDetails":{
            "type":"array",
            "description":"If multiple errors are reported they can be organized in a hierarchical structure in this array.",
            "items":[
                {
                    "$ref":"#/definitions/errorResource"
                }
            ]
        },
        "errorPath":{
            "type":"string",
            "description":"XPath or JSON path to indicate where the error occurs."
        },
        "errorCode":{
            "type":"string",
            "description":"Application specific error code, related to the error message shown at tittle."
        }
    }
}
嵌套架构：errorDetails
类型： array
如果报告了多个错误，则可以在此阵列中以分层结构组织它们。
显示来源
  • 数组：对象 errorResource

    "type":"array",
    "description":"If multiple errors are reported they can be organized in a hierarchical structure in this array.",
    "items":[
        {
            "$ref":"#/definitions/errorResource"
        }
    ]
