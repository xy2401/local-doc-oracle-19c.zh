1.170 LOG_ARCHIVE_CONFIG

LOG_ARCHIVE_CONFIG启用或禁用向远程目标发送重做日志和接收远程重做日志，并为Data Guard配置中的每个数据库指定唯一数据库名称（ DB_UNIQUE_NAME ）。

属性 描述

参数类型

串

句法

LOG_ARCHIVE_CONFIG =

 {
 [ SEND | NOSEND ]
 [ RECEIVE | NORECEIVE ]
 [ DG_CONFIG=(remote_db_unique_name1 [, ... remote_db_unique_name30) | NODG_CONFIG ]
 }

默认值

空值

可修改

改变系统

可在PDB中修改

没有

基本

没有

值

  • 发送

    允许将重做日志发送到远程目标

  • NOSEND

    禁用将重做日志发送到远程目标

  • 接收

    允许接收远程归档的重做日志

  • NORECEIVE

    禁用远程归档重做日志的接收

  • DG_CONFIG

    为Data Guard配置中的所有数据库指定最多30个唯一数据库名称（使用DB_UNIQUE_NAME初始化参数定义）的列表。

  • NODG_CONFIG

    消除先前使用DG_CONFIG选项指定的服务提供者名称列表。

如果未设置此参数，则其默认值为NULL 。但是， NULL值被视为参数已设置为'SEND, RECEIVE, NODG_CONFIG' 。
