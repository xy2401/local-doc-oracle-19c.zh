8.3从SQL DML调用Java

如果将Java方法作为函数发布，则可以从SQL SELECT ， INSERT ， UPDATE ， DELETE ， CALL ， EXPLAIN PLAN ， LOCK TABLE和MERGE语句中调用它们。例如，假设以下Java类的可执行文件存储在Oracle数据库中：

public class Formatter
{
  public static String formatEmp (String empName, String jobTitle)
  {
    empName = empName.substring(0,1).toUpperCase() +
                                     empName.substring(1).toLowerCase();
    jobTitle = jobTitle.toLowerCase();
    if (jobTitle.equals("analyst"))
      return (new String(empName + " is an exempt analyst"));
    else
      return (new String(empName + " is a non-exempt " + jobTitle));
  }
}

Formatter类具有formatEmp()方法，该方法返回包含职员姓名和工作状态的格式化字符串。编写此方法的调用规范，如下所示：

CREATE OR REPLACE FUNCTION format_emp (ename VARCHAR2, job VARCHAR2)
RETURN VARCHAR2
AS LANGUAGE JAVA
NAME 'Formatter.formatEmp (java.lang.String, java.lang.String)
return java.lang.String';

现在，调用format_emp函数来格式化员工列表：

SQL> SELECT format_emp(first_name, job_id) AS "Employees" FROM employees
  2   WHERE job_id NOT IN ('AC_MGR', 'AD_PRES') ORDER BY first_name;

限制

Java方法必须遵守以下规则，这些规则旨在控制副作用：

  • 从SELECT语句或并行INSERT ， UPDATE或DELETE语句调用方法时，该方法无法修改任何数据库表。

  • 从INSERT ， UPDATE或DELETE语句调用方法时，该方法无法查询或修改由该语句修改的任何数据库表。

  • 从SELECT ， INSERT ， UPDATE或DELETE语句调用方法时，该方法无法运行SQL事务控制语句，如COMMIT ，会话控制语句（如SET ROLE ）或系统控制语句（如ALTER SYSTEM 。另外，该方法不能运行数据定义语言（DDL）语句，例如CREATE ，因为它们后面是自动提交。

如果方法中的任何SQL语句违反上述任何规则，则在运行时会出现错误。
