12.26历史

句法

HIST[ORY] [n RUN | EDIT | DEL[ETE]] | [CLEAR | LIST]

允许用户从当前会话的历史记录列表中运行，编辑或删除以前使用的SQL * Plus，SQL或PL / SQL命令。您可以使用SET HISTORY命令在当前SQL * Plus会话中启用或禁用HISTORY命令。

HISTORY命令使您能够：

  • 列出命令历史列表中的所有条目。

  • 在命令历史列表中运行条目。

  • 编辑命令历史列表中的条目。

  • 从命令历史列表中删除条目。

  • 清除命令历史列表中的所有条目。

条款

HIST[ORY]

列出命令历史记录列表中的所有条目。 

n

表示命令历史记录列表中的条目。星号（*）表示命令历史列表中最后使用的命令。

RUN

使您可以从命令历史列表中执行条目n 。 

EDIT

使您可以使用默认文本编辑器在命令历史记录列表中编辑条目n 。在命令历史记录列表中编辑条目n并保存更改后，将在列表末尾创建一个新条目。当命令历史记录列表中的条目数达到最大限制时，列表中最旧的条目将被清除以容纳新条目。 

DEL[ETE]

使您可以从命令历史列表中删除条目n 。从历史记录列表中删除条目后，将重新排序列表以反映最新的更改。 

CLEAR

使您可以清除历史记录列表中的所有条目。清除后，无法恢复历史列表。 

LIST

列出历史列表中的所有条目。这与单独使用HIST [ORY]命令相同。

用法

您可以使用SQL * Plus DEFINE命令定义变量_EDITOR，以保存首选文本编辑器的名称。例如，要将EDIT使用的编辑器定义为vi，请输入以下命令：

DEFINE _EDITOR = vi

如果未定义_EDITOR，EDIT将尝试运行默认操作系统编辑器。有关更多信息，请参阅DEFINE命令。

例12-1例子

以下示例执行历史记录列表中的第五个条目：

HIST[ORY] 5 RUN

以下示例允许您编辑历史记录列表中的第三个条目：

HIST[ORY] 3 EDIT

以下示例允许您从历史记录列表中删除第二个条目：

HIST[ORY] 2 DEL[ETE]

以下示例允许您从历史记录列表中删除所有条目：

HIST[ORY] CLEAR

以下示例说明如何启用或禁用命令历史记录，以及如何检查命令历史记录状态：

SQL> set history on
SQL> show history
History is ON and set to "100"
SQL> set history off
SQL> show history
History is OFF
SQL> set history 1000
SQL> show history
History is ON and set to "1000"

以下示例显示如何列出历史记录列表中的所有条目：

SQL> show history
History is ON and set to "100"
SQL> show user
USER is "SYSTEM"
SQL> desc dual
 Name                                      Null?    Type
 ----------------------------------------- -------- ----------------------------
 PROD                                               VARCHAR2(1)

SQL> select * from dual;

D
-
X

SQL> history 4 run

D
-
X

SQL> history
  1  show history
  2  show user
  3  desc dual
* 4  select * from dual;

星号（*）表示命令历史列表中最后使用的命令。

以下示例说明如何列出历史记录列表中的所有条目，然后执行第二个条目：

SQL> history
  1  show history
  2  show user
  3  desc dual
* 4  select * from dual;
SQL> history 2 run
USER is "SYSTEM"
SQL> history
  1  show hist
* 2  show user
  3  desc dual
  4  select * from dual;
