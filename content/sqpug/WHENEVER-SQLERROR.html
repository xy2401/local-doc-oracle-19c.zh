12.53 WHERNVER SQLERROR

句法

WHENEVER SQLERROR {EXIT [SUCCESS | FAILURE | WARNING | n | variable  | :BindVariable] [COMMIT | ROLLBACK] | CONTINUE [COMMIT | ROLLBACK | NONE]}

如果是SQL，则执行指定的操作（默认情况下退出SQL * Plus） 命令或PL / SQL块生成错误。

条款

 [SUCCESS | FAILURE | WARNING | n | variable | :BindVariable]

指示SQL * Plus在检测到SQL命令或PL / SQL块错误时（但在打印错误消息后）立即执行指定的操作。SQL * Plus不会退出SQL * Plus错误。

EXIT [SUCCESS | FAILURE | WARNING | n | variable | :BindVariable]

一旦检测到SQL命令或PL / SQL块错误（但在打印错误消息后），就指示SQL * Plus退出。SQL * Plus不会退出SQL * Plus错误。WHENEVER SQLERROR的EXIT子句遵循与EXIT命令相同的语法。有关更多信息，请参阅EXIT 。 

CONTINUE

关闭EXIT选项。 

COMMIT

指示SQL * Plus在退出或继续之前执行COMMIT并将挂起的更改保存到数据库。 

ROLLBACK

指示SQL * Plus在退出或继续之前执行ROLLBACK并放弃对数据库的挂起更改。 

NONE

指示SQL * Plus在继续之前不执行任何操作。

用法

WHENEVER SQLERROR命令由SQL命令或PL / SQL块错误触发，而不是由SQL * Plus命令错误触发。

例子

以下脚本中的命令导致SQL * Plus退出并返回SQL错误代码 SQL UPDATE命令失败：

WHENEVER SQLERROR EXIT SQL.SQLCODE
UPDATE EMP_DETAILS_VIEW SET SALARY = SALARY*1.1;

以下示例显示WHENEVER SQLERROR命令在SQL * Plus命令出错后未执行，但如果SQL命令或PL / SQL块导致错误，则执行该命令：

WHENEVER SQLERROR EXIT SQL.SQLCODE
column LAST_name headIing "Employee Name"
Unknown COLUMN option "headiing"

SHOW non_existed_option

以下PL / SQL块错误导致SQL * Plus退出并返回SQL错误代码：

WHENEVER SQLERROR EXIT SQL.SQLCODE
begin
  SELECT COLUMN_DOES_NOT_EXIST FROM DUAL;
END;
/
