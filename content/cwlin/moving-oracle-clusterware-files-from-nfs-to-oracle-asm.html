将Oracle Clusterware文件从NFS移动到Oracle ASM

您可以将Oracle Cluster Registry（OCR）和投票文件从网络文件系统（NFS）移动到Oracle自动存储管理（Oracle ASM）磁盘组。

  1. 作为Oracle Grid Infrastructure安装所有者（ grid ），使用ASMCA创建Oracle ASM磁盘组。
    ./asmca

    按照ASMCA向导中的步骤创建Oracle ASM磁盘组，例如DATA 。

  2. 作为grid用户，将表决文件移动到您创建的Oracle ASM磁盘组：
    crsctl replace votedisk +DATA

    该命令的输出如下：

    CRS-4256: Updating the profile
Successful addition of voting disk 24c6d682874a4f1ebf54f5ab0098b9e4.
Successful deletion of voting disk 1b5044fa39684f86bfbe681f388e55fb.
Successfully replaced voting disk group with +DATA_DG_OCR_VDSK.
CRS-4256: Updating the profile
CRS-4266: Voting file(s) successfully replaced
  3. 作为grid用户，检查Oracle Cluster Registry（OCR）状态：
    ./ocrcheck

    该命令的输出如下：

    Status of Oracle Cluster Registry is as follows :
         Version                  :          4
         Total space (kbytes)     :     409568
         Used space (kbytes)      :       1380
         Available space (kbytes) :     408188
         ID                       :  288871063
         Device/File Name         : /oradbocfs/storage/12101/ocr
         Device/File integrity check succeeded
         Cluster registry integrity check succeeded
  4. 以root用户身份将OCR文件移动到您创建的Oracle ASM磁盘组：
    ./ocrconfig -add +DATA
  5. 以root用户身份从NFS位置删除Oracle Clusterware文件：
    ./ocrconfig -delete ocr_file_ path_previously_on_nfs
